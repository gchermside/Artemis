Introduction

Welcome to the CS2 HTML Tutorial. HTML stands for HyperText Markup Language, and despite its numerous revisions, it remains one of the most accessible markup languages, and certainly the most popular one among all available alternatives. It gives one the power to create structured multimedia documents with relative ease, and offers a variety of tags to achieve a wide range of page designs.

This tutorial will discuss tags up to and including those covered by the HTML 3.2 specifications. The HTML 4.0 specs, popularly known as DHTML (for Dynamic HyperText Markup Language) have given rise to much debate due to the fact that Netscape and Microsoft, makers of the two most popular web browsers, have declared completely different and incompatible specifications. This means that a page displaying perfectly in one browser may not even load in another. Although matters have recently been resolved with both companies agreeing to adhere to W3C's HTML 4.0 guidelines, it will take some time for these new specifications to gain widespread acceptance. For now, stick with us, and we promise not to blow up your browser.

General Structure

The structure of an HTML document resembles the following:

<HTML>

<HEAD>
<TITLE>The title of your document goes here.</TITLE>
</HEAD>

<BODY>
Your HTML code goes here.
</BODY>

</HTML>

Certain tags are interpreted correctly by one or both of Netscape Navigator 3.0+ and Internet Explorer 3.0+ (heretofore NN and IE) even if the tags are not included between the <BODY> and </BODY> tags. Those cases, however, constitute exceptions rather than the rules; therefore, you should keep your HTML code in the BODY of your document.

One notable exception to this rule is Javascript, which may sound somewhat familiar. Javascript has the popular reputation of being a secret discipline exercised by underground HTML cults as a way to write inexplicably cool web pages. In reality, it is a simple object-oriented programming language integrated into both NN and IE, one which allows the HTML author to include programming elements in any web page. Such elements may range from simple functionalities, such as popping up an alert box when the pointer is moved over a picture, to complex programs, such as games or text editors. Javascript function declarations belong between the <HEAD> and </HEAD> tags.

The Body Tag

The innocent looking <BODY> tag can be used to give a whole new look to your HTML document. When declared simply as <BODY>, it usually comes as default with black text on a gray background, with unvisited links in blue, visited links in purple and active links in red. However, an HTML document can override the defaults to declare an original color scheme. The <BODY> tag can accept the following attributes:

  • The ALINK attribute defines the active link color - the color that hyperlinks take on when clicked. The color value is represented as an RGB Triplet.
  • The BACKGROUND attribute declares a tiled background image. Both .gif and .jpg formats are supported.
  • The BGCOLOR attribute declares a solid background color. The color value is represented as an RGB triplet.
  • The LINK attribute sets the color of the hyperlinks throughout the page. Hyperlinks are special text strings which take you to a new page when clicked. The color value is represented as an RGB Triplet.
  • The TEXT attribute sets the color of the normal text throughout the page. The color value is represented as an RGB Triplet.
  • The VLINK attribute sets the color of the visited links throughout the page. A visited link constitutes a hyperlink which has already been clicked. The color value is represented as an RGB Triplet.

For instance, the following <BODY> tag,

<BODY BGCOLOR="#000000" TEXT="#30ff3d" LINK="#f89400" ALINK="#FF8DDD" VLINK="#B53CFF">

declares a page with background color in black, text in fluorescent green, hyperlinks in orange, active links in pink and visited links in purple.

Since figuring out RGB triplets by hand isn't much fun (trust us, it isn't) you probably want to use a utility which will translate colors into cryptic hexadecimal triplets for you. Such freeware utilities are widely available on the net.

How to Tag Your HTML Content

A successful presentation of your HTML content may be as important as the content itself, since the design will define the content's accessibility and appeal. The following tags will help you structure your document, placing text and graphics where they truly belong.

<P>Your text goes here.</P>

The paragraph tag defines a new paragraph, and all the text that goes between <P> and </P> appears as one whole block of text. HTML does not indent paragraphs. Paragraphs are aligned to the left by default, the ALIGN attribute which can take on the values RIGHT, CENTER or LEFT can be added to the <P> tag to dictate how the paragraph will be placed on the page. For instance,

<P ALIGN="RIGHT">blah blah blah</P>
would create a right-indented paragraph consisting of the text "blah blah blah". 

<A HREF="http://www.your.hyperlink.goes.here.com/">Your hyperlink goes here.</A>

The hypertext reference anchor tag creates a hyperlink - a special text string that takes you to a new URL when it`s clicked. The specified URL can be a hypertext document address, in the form of http://computername.domain.com/folder/filename.html, or an email address, in the form of mailto:email_address@host.com.

One important attribute of the anchor tag is NAME, which allows you to define a "named anchor" at an arbitrary point in the page. This allows you to link to different points in the page, as the index for this tutorial does. To define a named anchor, enclose the text to link to with the <a name="anchor_name"> and </a> tags. To link to this anchor, use the # sign followed by anchor_name, as in <a href="#anchor_name">Go to the named anchor</a>. If the named anchor is on another page, specify the filename before the # sign, as in <a href="filename.html#named_anchor">.

<IMG>

The image tag is used for the purpose of embedding images in an HTML document. The most important attributes for this tag are as follows:

  • The SRC attribute defines the source embedded in the document, such as the name of a .gif or .jpg image. Note that you must provide the full path to the image within the tag; that is, if you have the image barney.gif in a directory called dinos, then your tag would look like <IMG SRC="dinos/barney.gif">. This attribute is required.
  • The ALT attribute defines the alternative text that will be shown in the place of the picture in text-only browsers, in browsers which do not auto load images, or in any browser while the image is still loading. Recent browsers display the contents of the ALT attribute as the tool-tip-text for already loaded images. The ALT attribute should always be used to ensure usability for text-only browsers.
  • The BORDER attribute defines the thickness of the border around an image, in pixels.
  • The ALIGN attribute has about a googolplex possible values, allowing you to precisely place the image with respect to the other objects within the document. Some of these values are LEFT, RIGHT, TOP, TEXTTOP, MIDDLE, ABSMIDDLE, BASELINE, BOTTOM and ABSBOTTOM.
  • The WIDTH and HEIGHT attributes define the width and height of the picture, in pixels. Declaring these helps the browser allocate space for the picture ahead of time, therefore lowering download times.
  • The VSPACE and HSPACE attributes specify vertical and horizontal margins around the image, in pixels. 

<EMBED>

The embed tag can be used to embed multimedia content in an HTML document: Sound, music, video, VRML and so on. The embedded multimedia document is declared through the use of the SRC attribute.

<B>Your text goes here.</B>

The bold tag does exactly what its name suggests - it forms bold text.

<I>Your text goes here.</I>

The italic tag, much like its overweight sibling bold, modifies the text - this time italicizing it.

<U>Your text goes here.</U>

More elementary text modification antics. The underline tag, not surprisingly, underlines the text.

<S>Your text goes here.</S>

The strike-through tag draws a horizontal line across your text and provides a crossed-out effect.

NOTE: The above "physical" tags, although fully supported by HTML 3.2, have so-called "logical" alternatives. HTML 3.2 suggests using <STRONG> instead of <B>, <EM> instead of <I> and <STRIKE> instead of <S>. These alternatives are functionally identical in NN and IE. Using logical tags increases the clarity of your HTML code and provides a semantic meaning to the enclosed text, while using physical tags emphasizes a particular style and ensures better compatibility with more browsers.

<FONT>Your text goes here.</FONT>

The font tag is utilized mainly to make use of its attributes, which can do weird and wonderful things to the text in your HTML document.

  • The SIZE attribute can take on values 1 through 7, which correspond to font sizes 8, 10, 12, 14, 18, 24 and 36, as well as +value or -value, to modify the current font size. For instance, if your browser has font size 12 (SIZE=3) set as default, and if you declare <FONT SIZE="-2"> at the beginning of your HTML document, the text until the </FONT> tag will appear at font size 8 (3-2=1, therefore SIZE="1"). However, if you simply declare <FONT SIZE="1">, you know that you are going to get font size 8, no matter what the browser's settings may be.
  • The FACE attribute takes on font names as values. It's usually a good idea to list a few font names, so that the browser will go through them in order and use the first font that it finds on the user's system. For instance, if you declare <FONT FACE="Comic Sans MS,Footlight MT Light,Times New Roman">, your browser will try to find the first font on your system; it will start searching for the second one if the first one is not there; and failing that, it will resort to displaying the document in Times New Roman. Although this attribute is supported both by NN and IE, it is not a part of HTML 3.2.
  • The COLOR attribute, much like the same attribute for the tag, modifies the color of the text. The color value is represented as an RGB triplet. 

<H1>Your text goes here.</H1>

The heading tag, which ranges from <H1> to <H6>, largest to smallest, defines a heading - a bold title. Although heading tags can be achieved through the use of the <FONT> and <B> tags, they provide the semantic meaning that the enclosed text is a heading rather than normal text. The heading is aligned to the left by default, the ALIGN attribute, which can take on the values RIGHT, CENTER or LEFT, can be added to modify the placement of the text.


<UL>
<LI>Item</LI>
<LI>Another item</LI>
</UL>

The unordered list tag creates a list of bullet points. Each separate bullet point is placed between a pair of <LI> and </LI> tags. An ordered list can be created in exactly the same way, by specifying <OL> ... </OL> rather than <UL> ... </UL>. The OL tag supports the attribute TYPE with possible values A (A, B, C ... ), a (a, b, c ... ), I (I, II, III ... ), i (i, ii, iii ... ) as well as the default (1, 2, 3 ... ). It also supports the attribute START which declares the number that the list begins with.

<HR>

The <HR> tag displays a nice horizontal line on the screen. It is useful for visually separating different sections of your document. You can define the width of the line using the WIDTH attribute. The WIDTH can be specified as a pixel value (e.g. 100) or a percentage of the screen (e.g. 50%). You can also define the lines thickness in pixels using the SIZE attribute, its alignment using the ALIGN attribute (possible values are LEFT, RIGHT, CENTER with CENTER being default), and finally its 3D look with the NOSHADE attribute ( possible values are YES or NO).

Tables

Tables can be extremely useful in defining a grid within which you can place various objects, such as blocks of text, images, sound, or video. A table is placed between a pair of <TABLE> ... </TABLE> tags. Some of the most important attributes supported by the <TABLE> tag are as follows:

  • The WIDTH and HEIGHT attributes declare the width and height of the table, either in pixels or as a percentage of the size your browser window. For instance, <TABLE WIDTH="50%"> declares a table that is half as wide as your browser window. Resizing the window automatically resizes the table.
  • The ALIGN attribute, much like its <P> counterpart, takes on the values LEFT, RIGHT and CENTER with LEFT as default. It specifies the placement of the table within the browser window.
  • The VALIGN attribute specifies the placement of the data within table data cells. It takes on the values TOP, BOTTOM, BASELINE or CENTER with CENTER as default.
  • The BGCOLOR fulfills the same function that its counterpart performs, specifying a solid background color for the table.
  • The BORDER tag defines the thickness of the table border, the CELLPADDING tag defines the space between the cell data and cell wall., and the CELLSPACING tag defines the space between cells, all in pixels.

Once you declare a table, you must declare each table row separately by using a pair of <TR> ... </TR> tags. You must then declare the columns within a row as distinct table data cells by using a pair of <TD> ... </TD> tags. Table data cells must be nested within their respective table rows, which must in turn be nested within their respective table.

<TD> and <TR> tags can take on all the attributes supported by <TABLE>. An attribute declared by a specific <TD> overrides the same attribute declared by its parent <TR>, which in turn overrides the same attribute declared by its parent <TABLE> .

Additional attributes supported by <TD> are the following:

  • The NOWRAP attribute takes on the values YES or NO. It dictates whether cell data wraps to fit the width of the cell.
  • The COLSPAN and ROWSPAN attributes specify how many columns and rows the data item spans, in respective order. The default for both attributes is 1. Note that each row must have the same number of cells, although the number of <TD> tags you nest within a pair of <TR> ... </TR> tags defines how your table data items use those particular cells. With proper use of the COLSPAN and ROWSPAN attributes, you can get a single data item (a block of text, an image etc...) to stretch across multiple cells, horizontally or vertically.

Check out the following HTML code, and the table that it creates:

<TABLE CELLPADDING="5" CELLSPACING="10" BORDER="2" WIDTH="100" HEIGHT="100">
	<TR ALIGN ="CENTER">
		<TD><STRONG>Item One</STRONG></TD>
		<TD BGCOLOR="#888fff">Item Two</TD>
	</TR>
	<TR BGCOLOR="#ffaaff">
		<TD COLSPAN="2">Item Three, plus a description</TD>
	</TR>
</TABLE>

Forms

Forms allow the user to enter information into a series of fields, such as text fields, radio buttons, checkboxes, and pull-down menus. This information can then be submitted either to an email address, or more regularly, to a CGI script that processes this information. Forms are comprised of one or more input objects (text boxes, pull down menus, etc.) all placed within the <FORM> tag. There are no special rules for where the objects in a form should be layed out by a browser, so you should use tables or style sheets to control the look of your form.

In order to send the information entered in a form to a given URL, the <FORM> ... </FORM> tag can take on the ACTION attribute, which specifies either the URL or e-mail address to which the data will be submitted.

ex. <form action="mailto:John_Doe@Brown.edu">

In order to initialize the information that will be sent to an e-mail address by the form into a readable format, the ENCTYPE attribute is used.

ex. <form action="mailto:John_Doe@Brown.edu" enctype="text/plain">

Upon submitting a form, it is possible for you to alert the user that the form is being submitted by using the ONSUBMIT command.

ex. <form action="mailto:John_Doe@Brown.edu" enctype="text/plain" onSubmit="window.alert('This form is being sent by e-mail')">

Forms also supports the METHOD attribute, which can take on the values GET or POST. These commands are required when sending information to an URL for CGI processing. POST is generally better to use when the form is sent to an email address rather than a URL.

The following tags must all be nested between a pair of <FORM> ... </FORM> tags in order to achieve their desired results:

The <INPUT> tag constitutes the most important form element. It defines most common form "controls" like text-fields, multiple-choice lists, and submission buttons. Only the TYPE and NAME attributes are required for each element in an <INPUT> tag.

The TYPE attribute defines what type of element will be used for input (button, text, etc.). The NAME attribute defines what the name of the form element will be. Below are some common TYPE attributes:

  • A TEXT field is a simple, one-line field.
  • A CHECKBOX allows more than one box to be selected at a time.
  • A RADIO button lets the user select one value among several options.
  • The SUBMIT button sends in a form with the current field values.
  • The RESET button sets all the fields back to their default values.

The VALUE attribute defines the default value for the field (i.e. the name of a button, or intial textbox text). This attribute is particularly important when defining the label for a button.

ex. <input type=submit value="Contact WWW-Wizard">

The SIZE and MAXLENGTH determine the attributes of a TEXT element. Size is the initial length of the textbox, and maxlength is how big it should get.

ex. <input type=text name=age size=8 maxlength=20>

The <SELECT> ... </SELECT> tag specifies a pull-down menu. It can take on a NAME attribute which, like its counterpart in the INPUT tag, specifies the name of the field being declared. A pull-down menu contains numerous options, each of which is defined by an tag. The OPTION tag takes on a VALUE attribute, which defines the name of the field being declared. Each text string that appears in the pull-down menu goes between a pair of <OPTION> ... </OPTION> tags. All <OPTION> ... </OPTION> tags must be nested within a pair of <SELECT> ... </SELECT> tags.

ex.
<select name=class size=2 multiple>
<option> Junior
<option> Senior
</select>

The <TEXTAREA> ... </TEXTAREA> tag defines a scrollable, multi-line text field where the user can input ASCII text. It can take on a NAME attribute, which defines the field name. The default value for the textarea is the text in between the <TEXTAREA> and </TEXTAREA> tags. The number of rows and columns can be defined through the use of ROWS and COLS attributes. Finally, the WRAP attribute, which can take on the values PHYSICAL or VIRTUAL, automatically wraps text around when the end of the text area is reached.

Frames

The use of frames is cause for much debate among HTML experts. While some claim that frames can be a powerful tool for navigation and for factoring out code from web pages, others contend that frames are inherently ugly and should be executed on the spot. The following will give you an overview of how frames work and what they are capable of.

The <FRAMESET> ... </FRAMESET> tags replace the <BODY> ... </BODY> tags within an HTML document. They define a frameset within which HTML documents appear. The FRAMESET tag can take on the attributes COLS or ROWS, each of which can specify the number and sizes of the frames to be included. Sizes can be specified as pixel values or as percentages. A wildcard (*) can be used at most once in either attribute to denote that the frame will take up whatever space remains on the screen. The <FRAME> tag specifies the name and source file of a particular frame. The SRC attribute defines the HTML file that will appear within that frame. The NAME attribute declares a name for the frame. This name can later be used as the target of a hyperlink, making the document specified by the hyperlink appear in that frame. Finally, the FRAME tag can accept the SCROLLING attribute with the values YES, NO, or AUTO and the NORESIZE attribute with the values YES or NO. Frames require the hypertext reference anchor tag to be used somewhat differently. When declaring an <A HREF> tag, apart from specifying the URL reference, it is necessary to also provide a TARGET attribute with a value referring to the name of the frame being targeted. For instance, in a FRAMESET that includes frames named navigation and body, an HTML document called target.html would be displayed in the body frame by clicking on a hyperlink defined as follows: 
<A HREF="target.html" TARGET="body">Why don't you click here?</A>

Text included between a pair of <NOFRAMES> ... </NOFRAMES> tags ensures that all users will receive some form of HTML content. The text between a pair of ... tags will be ignored by frames-enabled browsers, but it will be displayed by all other browsers. NN and IE are both frames-enabled.

For instance, check out the following HTML code, which creates a column that is 20% as wide as the browser window on the left side, a 200-pixel wide column on the right side, and allocates the remaining space to the middle column. The leftmost column is then sliced into two rows, the top row being 250 pixels high and the bottom row inheriting whatever space is left of the column. If this vivid description has not satisfied you, see for yourself.

<FRAMESET COLS="20%,*,200">
	<FRAMESET ROWS="250,*">
		<FRAME SRC="a.html" NAME="A">
		<FRAME SRC="b.html" NAME="B">
	</FRAMESET>
	<FRAME SRC="c.html" NAME="C">
	<FRAME SRC="d.html" NAME="D">
</FRAMESET>
